## Email Components

| Component | Description |
|-----------|-------------|
| **To** | Primary recipient's email address |
| **CC** | Carbon Copy — secondary recipients (all can see) |
| **BCC** | Blind Carbon Copy — hidden recipients (others can't see) |
| **Subject** | Brief description of email topic |
| **Body** | **Main message content** |
| **Attachment** | File sent with email |
| **Signature** | Auto-added text at end (name, contact) |

---

## Email Protocols

| Protocol | Full Form | Purpose |
|----------|-----------|---------|
| **SMTP** | Simple Mail Transfer Protocol | **Sending** email |
| **POP3** | Post Office Protocol 3 | **Receiving** — downloads to device |
| **IMAP** | Internet Message Access Protocol | **Receiving** — stays on server |

### POP3 vs IMAP

| Feature | POP3 | IMAP |
|---------|------|------|
| Storage | Downloads to device | Stays on server |
| Multiple devices | Difficult | Works well |
| Offline access | Yes (after download) | Limited |
| Best for | Single device | Multiple devices |

## Email Security

| Term | Meaning |
|------|---------|
| **Encryption** | **Protects message confidentiality** — unreadable to others |
| **Spam** | Unwanted/junk email |
| **Phishing** | Fake emails to steal personal information |
| **Spoofing** | Fake sender address |
| **Spam filter** | Automatically moves spam to junk folder |
| **Two-factor authentication** | Extra verification for email login |

## Email Routing

| Process | Description |
|---------|-------------|
| How email travels | **Through a series of mail servers** |
| Sender → SMTP server → DNS lookup → Recipient's mail server → Inbox |
| Mail server | Computer that sends/receives/stores email |
| MTA | Mail Transfer Agent — moves email between servers |

## Exam Traps ⚠️
> Body of email = **main message content** — NOT subject line or attachment
> SMTP = **sending** — POP3/IMAP = **receiving** — never mix these
> Encryption = **confidentiality** — NOT faster delivery or larger file size
> Email routing = **through mail servers** — NOT direct computer to computer
> BCC = **hidden from others** — CC = everyone can see who received
> Phishing = **fake email to steal info** — NOT just spam/junk mail
> POP3 = **downloads** email; IMAP = **keeps on server**
